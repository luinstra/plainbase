plainbase Design System
Light Dark shown in parallel

The look should grow from the logo, not ignore it.

The current build is a solid skeleton, but its personality is generic: a blue accent and system fonts that could belong to any tool. Meanwhile the strongest asset you already made — the wordmark — is going unused as a design language.

So this system does one thing deliberately: it takes the logo's teal slash and its JetBrains Mono voice and lets them set the rules for everything else. Calm, content-first, and honest about being made of plain files.

The five moves
1Teal is the accent. The logo's teal replaces blue everywhere a person points, hovers, or lands.
2Mono gets a job. JetBrains Mono becomes the voice of structure — paths, tags, status, keys, code — echoing the wordmark.
3IBM Plex Sans for humans. One warm, technical sans for all UI and reading. (No Inter.)
4The slash is a motif. The skewed teal mark signals "you are here" — active nav, breadcrumbs, callouts.
5Restraint by default. Type and spacing carry the design; decoration is rare and earned.
